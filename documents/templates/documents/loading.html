국제 전기 전자 공학회 IEEE (Institute of Electrical and Electronics Engineers)

Protocol

여러 컴퓨터나 단말기 사이에서 데이터 통신을 신뢰성 있고 원활하게 하기 위해 필요한 통신 규약

송신자측에서 수신자측을 호출하여 연결하는 과정, 통신 회선을 통해 전달되는 정보의 형태, 오류발생에 대한 제어, 송/수신자측 간의 동기방식 등에 대한 약속을 포함한다.

OSI MODEL

네트워크 통신을 체계적으로 다루고 있는 ISO 표준은 개방 시스템 상호연결 모델이다.

개방 시스템은 기반 구조와 관계 없이 시스템 간의 통신을 제공하는 프로토콜 집함이다. OSI 모델은 하드웨어나 소프트웨어 기반의 논리적인 변화에 대한 요구 없이 서로 다른 시스템 간의 통신을 원활하게 하는데 목적이 있다.

국제표준화기구 ISO ( International Organization for Standardization )

FTP(File Transfer Protocol)

: 인터넷 상에서 파일을 전송하기 위한 프로토콜로, 클라이언트와 서버 간에 파일 전송을 지원한다. 일반적으로 파일 업로드, 다운로드, 파일 및 디렉터리 생성, 수정, 삭제 등의 작업을 수행한다.

SSH(Secure Shell)

: 네트워크 프로토콜 중 하나로, 안전한 원격 접속을 가능하게 한다. Telnet과 달리 암호화된 터미널 세션을 제공하여, 사용자의 비밀번호나 데이터를 안전하게 전송할 수 있다. SSH는 원격 로그인, 파일 전송, 명령어 실행 등 다양한 기능을 제공한다.

DNS(Domain Name System)

: 인터넷 상에서 도메인 이름과 IP 주소를 변환하는 시스템이다. DNS는 사용자가 입력한 도메인 이름을 IP 주소로 변환하여, 해당 웹 서버에 접속할 수 있도록 한다. DNS는 전 세계적으로 분산되어 운영되며, 인터넷 상에서 가장 중요한 인프라 중 하나이다.

TCP(Transmission Control Protocol)

: 전송 계층 프로토콜 중 하나로, 신뢰성 있는 데이터 전송을 지원한다. TCP는 데이터 전송을 위해 연결 지향적인 프로토콜로, 3-way handshake를 통해 연결 설정 후, 데이터 전송을 시작한다. 데이터를 패킷 단위로 분할하여 전송하며, 수신 측에서는 순서를 재조합하고 에러 검출 및 복구를 수행한다.

UDP(User Datagram Protocol)

: 전송 계층 프로토콜 중 하나로, 비신뢰적인 데이터 전송을 지원한다. UDP는 연결 설정 과정이 없고, 데이터를 패킷 단위로 전송한다. 데이터 전송의 신뢰성보다는 빠른 속도를 중시하며, 에러 검출과 복구는 수행하지 않으며 게임, 스트리밍 등에서 실시간 데이터 전송을 위해 사용된다.

TCP와 UDP의 차이점

TCP와 UDP의 차이점은 신뢰성과 속도이다. TCP는 데이터 전송의 신뢰성을 보장하기 위해 많은 오버헤드가 발생하여 전송 속도가 느리지만, 에러 검출과 복구를 수행하여 데이터의 신뢰성을 보장한다. 반면, UDP는 전송 속도가 빠르지만, 에러 검출과 복구를 수행하지 않으므로 데이터의 신뢰성이 떨어질 수 있다.

즉, TCP는 파일 전송, 이메일 등의 신뢰성이 중요한 데이터 전송에 적합하며, UDP는 실시간 데이터 전송에 적합하다.
